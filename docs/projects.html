🎮 Behavioral Playtime Analytics: The PLAICraft Project

UBC Computer Science Research Group Collaboration

Overview

Analyzed player acquisition and engagement data generated by users on PLAICraft, a custom Minecraft server deployed by a Computer Science research group at the University of British Columbia. The objective was to determine specific player demographic profiles that maximize data contributions to improve strategic server recruitment efforts.

Core Analytics & Methodology

  • Feature Engineering: Isolated individual player demographic attributes and total tracked playtime segments out of multi-relational gaming telemetry logs.
  • Predictive Modeling: Built a regression model mapping individual age factors against player session lengths.
  • Strategic Optimization: Identified an engagement inflection point highlighting that university-aged players under 28 yielded the highest data densities, providing actionable targeting criteria for the research group.

Technical Stack

  • Language: R
  • Core Libraries: tidyverse, ggplot2, broom
  • Data Format: Gaming & Session Logs (.csv)

Primary Hub: UBC Undergraduates
Objective: Maximize data collection and engagement

🌎 NASA EONET API Natural Event Tracker

Programmatic REST API Consumption & JSON Geometry Flattening

Overview

Built an API pipeline to programmatically query NASA’s Earth Observatory Natural Event Tracker (EONET), isolating, cleaning, and categorizing global natural events—such as wildfires, storms, and volcanic activity—over time.

Core Analytics & Methodology

  • API Architecture: Authenticated and structured native HTTP GET requests to query NASA’s live REST API endpoint recursively.
  • Data Transformation: Parsed complex, deeply nested JSON spatial vectors and multi-dimensional geometry coordinates into clean, flat relational data frames.
  • Data Pipelines: Integrated robust error-handling workflows to handle empty objects or missing observation fields safely during extraction.

Technical Stack

  • Language: R
  • Core Libraries: httr, jsonlite, tidyverse, ggplot2
  • Data Format: JavaScript Object Notation (JSON)

📊 Web Scraping and HTML Parsing with rvest

E-Commerce Sandbox Data Extraction & Structure

Overview

Developed an automated data extraction engine to collect structured data from web pages using rvest and parse complex HTML elements for analysis.

Workflow

  • Ethical Compliance: Inspected and verified script alignment with the site’s robots.txt permissions to ensure responsible scraping practices.
  • DOM Parsing: Extracted semi-structured HTML components including item titles, dynamic prices, star ratings, and stock availability status.
  • Data Cleansing: Cleaned and unnested raw text strings into a highly structured, analysis-ready tibble dataframe.

Tech Stack

  • Language: R
  • Core Libraries: rvest, xml2, tidyverse
  • Data Format: Semi-structured HTML

Analytical Highlight: Data Governance & Privacy

  • Ethical Data Ownership: Evaluated licensing constraints, documented data provenance, and aligned the pipeline with FAIR data principles (Findable, Accessible, Interoperable, Reusable) to guarantee reproducible research.
  • Privacy-Preserving Analytics: Mitigated re-identification risks of sensitive attributes by implementing structural data anonymization techniques.
    • Generalization & Discretization: Transformed high-risk continuous numeric attributes into secure category bands.
    • Differential Privacy Foundations: Synthesized a random Normal distribution noise layer (\(~N(\mu, 1)\)) over specific metrics to mask individual records while maintaining underlying population trends.
    • Model Evaluation: Audited data structures against \(k\)-Anonymity and \(l\)-Diversity privacy compliance benchmarks.
